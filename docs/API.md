## Общая информация
- **Базовый URL:** `http://{host}:{port}` (по умолчанию `http://localhost:8000`)
- **Версия API:** `v1`
- **Префикс всех бизнес-эндпоинтов:** `/api/v1`
- **Аутентификация:** JWT Bearer. Действующий `access_token` передаётся в заголовке `Authorization: Bearer <token>`.
- Для основных бизнес-эндпоинтов подтверждение email **не требуется** (поддерживаются анонимные пользователи).

## Служебные эндпоинты

**Пример ответа 200**

```json
  "service": "InnerCore Backend API",
  "version": "1.0.0",

```

- **Аутентификация:** не требуется.

**Пример ответа 200**
```json
{
  "status": "ok",

  "version": "1.0.0"
}
```

## Аутентификация и управление аккаунтом (`/api/v1/auth`)

### `[NEW]` POST `/api/v1/auth/anonymous`
- **Назначение:** Анонимная авторизация по `device_id`.
- **Аутентификация:** не требуется.
- **Тело запроса:**
  ```json
  {
    "device_id": "uuid-string",
    "platform": "ios|android",
    "app_version": "1.0.0"
  }
  ```
- **Ответ 200:**
  ```json
  {
    "access_token": "jwt",
    "refresh_token": "jwt",
    "user": {
      "id": "uuid",
      "is_anonymous": true,
      "email": null
    }
  }
  ```

### `[NEW]` POST `/api/v1/auth/link`
- **Назначение:** Привязка Google/Apple к текущему пользователю.
- **Аутентификация:** требуется `access_token`.
- **Тело запроса:**
  ```json
  {
    "provider": "google|apple",
    "id_token": "provider_id_token"
  }
  ```
- **Ответ 200:**
  ```json
  {
    "linked": true,
    "user": {
      "id": "uuid",
      "is_anonymous": false,
      "email": "user@email.com"
    },
    "provider_identity": {
      "provider": "google",
      "provider_subject": "xxxx",
      "email": "user@email.com"
    }
  }
  ```
- **Основные ошибки:**
  - `400` — `invalid_provider`, `invalid_token`, `token_expired`, `audience_mismatch`.
  - `401` — нет/невалидный JWT.
  - `409` — `identity_already_linked`, `user_already_has_identity`.

### POST `/api/v1/auth/register`
- **Назначение:** Регистрация пользователя по email/паролю.
- **Аутентификация:** не требуется.
- **Тело запроса:**
  ```json
  {
    "email": "user@example.com",
    "password": "S3curePassw0rd",
    "first_name": "Carl",
    "last_name": "Jung",
    "timezone": "Europe/Moscow"
  }
  ```

**Основные ошибки**
- `400` — email уже занят.
- `422` — передан некорректный `timezone` (используйте IANA-названия, например `Europe/Moscow` или `UTC`).
- `500` — внутренняя ошибка при создании пользователя.

- **Назначение:** Удаление аккаунта текущего пользователя.

### POST `/api/v1/auth/logout`
- **`[INFO]`** Эндпоинт не несет практической пользы при stateless-архитектуре. Выход из системы реализуется на клиенте путем удаления токенов.
- **Назначение:** Логическое завершение сессии на клиенте (сервер статуса токена не хранит).
- **Аутентификация:** опционально.

## Сны (`/api/v1/dreams`)

### POST `/api/v1/dreams`
- **Назначение:** Создание нового сна.
- **Лимиты:** не более 5 записей в сутки (по часовому поясу пользователя).
- **Тело запроса:**
  ```json
  {
    "content": "Мне снилось, что я гуляю по тёмному сосновому лесу...",
    "title": "Сон про лес", // опционально
    "emoji": "🌲", // опционально
    "comment": "Показался важным" // опционально
  }
  ```

**Ответ 201**
```json
{
  "id": "d2d39523-2f6a-46a8-a2c8-1a0e41285672",
  "user_id": "44cb8dca-4b02-425d-ba33-48423ab2fd0f",
  "title": "Сон про лес",
  "content": "Мне снилось, что я гуляю по тёмному сосновому лесу...",
  "emoji": "🌲",
  "comment": "Показался важным",
  "recorded_at": "2025-11-11T08:20:11.123Z",
  "created_at": "2025-11-11T08:20:11.123Z",
  "updated_at": "2025-11-11T08:20:11.123Z",
  "has_analysis": false
}
```

**Основные ошибки**
- `429` — превышен лимит 5 снов в сутки.

---

### GET `/api/v1/dreams`
- **Назначение:** Пагинированный список снов пользователя.
- **Параметры запроса:**
  - `page` (по умолчанию `1`, `>=1`)
  - `page_size` (по умолчанию `10`, `1-100`)
  - `date` (опционально, `YYYY-MM-DD`)

- **Параметры:** `q` — строка поиска (обязательная, длина ≥ 1).

## Анализы (`/api/v1/analyses`)

> **`[INFO]`** Вся секция будет переработана для поддержки диалогового режима. Вместо одного результата анализа будет чат.

### POST `/api/v1/analyses`
- **Назначение:** Запросить **первичный** анализ для сна. Один сон → один анализ (одна комната чата).
- **Тело запроса:**
  ```json
  {
    "dream_id": "d2d39523-2f6a-46a8-a2c8-1a0e41285672"
  }
  ```
- **Ответ 202:**
  ```json
  {
    "analysis_id": "d82d0a6f-8d1d-4af9-8d7d-4d8f99cd18f5", // ID комнаты чата
    "task_id": "5cfb22a6-4f38-4b6e-95b7-0e8dbe7d5a45",
    "status": "pending",
    "message": "Analysis task created. Use analysis_id to access the chat."
  }
  ```

---

### GET `/api/v1/analyses/dream/{dream_id}`
- **Назначение:** Получить анализ по ID сна (статус/результат).
- **Ответ 200:**
  ```json
  {
    "id": "d82d0a6f-8d1d-4af9-8d7d-4d8f99cd18f5",
    "dream_id": "d2d39523-2f6a-46a8-a2c8-1a0e41285672",
    "status": "completed",
    "result": "Глубокий анализ сна..."
  }
  ```

---

### GET `/api/v1/analyses/task/{task_id}`
- **`[INFO]`** Остается актуальным для отслеживания статуса генерации **первичного** анализа.
- **Назначение:** Проверка статуса Celery-задачи.

## Заголовки и кодировки
- Все тела запросов/ответов JSON кодируются в UTF-8.
- Для multipart-загрузок используйте корректные MIME-типы (`image/jpeg`, `image/png`).
- При работе с Celery статусы задач регистрозависимы — клиенту следует обрабатывать `PENDING`, `STARTED/PROCESSING`, `SUCCESS/COMPLETED`, `FAILURE/FAILED`.

## Чат анализа (`/api/v1/messages`)

### POST `/api/v1/messages`
- **Назначение:** Отправить follow‑up сообщение в чат по сну.
- **Аутентификация:** требуется `access_token`.
- **Тело запроса:**
  ```json
  {
    "dream_id": "d2d39523-2f6a-46a8-a2c8-1a0e41285672",
    "content": "Расскажи подробнее про архетип тени."
  }
  ```
- **Ответ 202:**
  ```json
  {
    "task_id": "task-id",
    "status": "processing",
    "user_message": {
      "id": "msg-id",
      "dream_id": "d2d39523-2f6a-46a8-a2c8-1a0e41285672",
      "role": "user",
      "content": "Расскажи подробнее про архетип тени.",
      "created_at": "2025-11-11T09:16:10.000Z"
    }
  }
  ```

### GET `/api/v1/messages/dream/{dream_id}`
- **Назначение:** История чата по сну.
- **Параметры:** `limit` (1-200), `offset` (>=0).
- **Ответ 200:**
  ```json
  {
    "messages": [
      {
        "id": "msg1",
        "dream_id": "d2d39523-2f6a-46a8-a2c8-1a0e41285672",
        "role": "assistant",
        "content": "Глубокий анализ сна...",
        "created_at": "2025-11-11T08:25:45.012Z"
      }
    ],
    "total": 1
  }
  ```

### GET `/api/v1/messages/task/{task_id}`
- **Назначение:** Статус задачи генерации ответа LLM для follow‑up сообщений.
- **Аутентификация:** требуется `access_token`.
- **Ответ 200:**
  ```json
  {
    "task_id": "task-id",
    "status": "PENDING|PROCESSING|SUCCESS|FAILURE",
    "result": "optional",
    "error": "optional"
  }
  ```

## Примечания по безопасности
- Refresh-token хранится только на клиенте (серверного хранилища нет).
- Удаление аккаунта выполняет каскадное удаление связанных сущностей (сны, анализы, OAuth-аккаунты).
- Все критичные параметры (URL БД, ключи JWT, SMTP и т.д.) приходят из `.env` и должны быть настроены перед запуском.
