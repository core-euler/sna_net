JungAI — User Onboarding Modal Specification

Версия: 1.0
Статус: ТЗ для новой функции
Область: Flutter клиент + минимальная backend интеграция

Важно: этот документ описывает только систему многоступенчатого онбординга пользователя.
Он не изменяет другие части приложения.

⸻

1. Цель

Создать более качественный процесс первого знакомства пользователя с приложением.

Текущий онбординг состоит из одного текстового вопроса и выглядит как обычное поле ввода.

Это нужно заменить на структурированный многошаговый онбординг, который:
	•	собирает базовый психологический и жизненный контекст пользователя
	•	делает это мягко и ненавязчиво
	•	помогает улучшить дальнейший анализ снов

Все ответы пользователя в итоге сохраняются в одно поле about_me.

⸻

2. Когда показывается онбординг

Онбординг показывается только при первом входе пользователя.

Условия показа:

onboarding_completed == false

или

about_me == empty

После завершения онбординга:

onboarding_completed = true

Онбординг больше не показывается.

⸻

3. Формат UI

Онбординг не должен быть отдельным экраном.

Он должен открываться как модальное окно поверх главного интерфейса.

Требования:
	•	основной экран приложения остаётся на фоне
	•	фон приложения blurred
	•	модальное окно центрировано
	•	высота окна примерно 70–85% экрана
	•	скруглённые углы
	•	мягкая тень

Визуальное ощущение:

“разговор с приложением перед началом использования”.

⸻

4. Общая структура онбординга

Онбординг состоит из 5 последовательных шагов.

Каждый шаг:
	•	содержит один вопрос
	•	содержит поле ответа
	•	содержит кнопку Далее
	•	содержит кнопку Пропустить

Важно:

пользователь может пропустить любой шаг.

⸻

5. Шаги онбординга

Шаг 1 — Пол и возраст

Цель: базовая демография.

UI:
	•	выбор пола
	•	ввод возраста

Оба поля optional.

Пример текста:

“Чтобы лучше понимать контекст ваших снов, расскажите немного о себе.”

⸻

Шаг 2 — Род деятельности

Цель: жизненная среда пользователя.

Вопрос:

“Чем вы сейчас занимаетесь?”

Примеры подсказок:
	•	студент
	•	предприниматель
	•	работа в найме
	•	творческая деятельность

Тип ввода:

свободный текст.

⸻

Шаг 3 — Семья и отношения

Цель: социальный контекст.

Вопрос:

“Расскажите немного о вашей семье или близких людях.”

Подсказка:

“Например: живёте ли вы один, есть ли партнёр, дети или важные отношения.”

Тип ввода:

свободный текст.

⸻

Шаг 4 — Интересы и увлечения

Цель: понять личные интересы.

Вопрос:

“Какие у вас увлечения или интересы?”

Примеры:
	•	искусство
	•	спорт
	•	технологии
	•	путешествия

Тип ввода:

свободный текст.

⸻

Шаг 5 — Жизненный контекст

Цель: понять текущий период жизни пользователя.

Вопрос:

“Что сейчас происходит в вашей жизни или что для вас особенно важно в данный период?”

Подсказка:

“Вы можете рассказать о целях, трудностях, изменениях или поисках.”

Тип ввода:

многострочный текст.

⸻

6. Навигация

Внизу модального окна должны быть кнопки:
	•	Назад
	•	Далее
	•	Пропустить

Требования:
	•	пользователь может вернуться к предыдущему шагу
	•	любой шаг можно пропустить

На последнем шаге кнопка:

Завершить

⸻

7. Сохранение данных

Все ответы должны быть объединены в одно текстовое поле:

about_me

Пример формата:

Gender: male
Age: 34
Occupation: software developer
Family: married, one child
Interests: psychology, philosophy, hiking
Life context: currently changing career and exploring personal meaning

Если шаг пропущен — поле не добавляется.

После завершения онбординга данные отправляются на backend.

⸻

8. UX требования

Онбординг должен ощущаться как разговор, а не как анкета.

Поэтому:
	•	избегать формального языка
	•	использовать мягкие формулировки
	•	добавить короткие пояснения

Интерфейс должен быть:
	•	спокойный
	•	минималистичный
	•	психологически безопасный

⸻

9. Flutter архитектура

Рекомендуемая структура компонентов:

OnboardingModal
OnboardingStep
OnboardingController

State:

currentStep
answers

Рекомендуемый способ отображения:

showDialog()

или

showModalBottomSheet()

с кастомным layout.

⸻

10. Acceptance Criteria

Онбординг считается реализованным, если:
	1.	При первом запуске появляется модальное окно онбординга.
	2.	Фон приложения blurred.
	3.	Онбординг содержит 5 шагов.
	4.	Каждый шаг можно пропустить.
	5.	Есть навигация назад и вперёд.
	6.	Ответы объединяются в about_me.
	7.	Данные сохраняются на backend.
	8.	После завершения онбординг больше не появляется.

⸻
