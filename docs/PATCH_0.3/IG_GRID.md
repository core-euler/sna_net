# JungAI — UI Redesign Patch: Instagram-style Grid

Версия: 1.0
Статус: ТЗ на редизайн клиентской части
Область: только Flutter UI
Цель: перевести главный экран списка снов из вертикального списка в плиточную сетку в стиле Instagram.

---

## 1. Scope

В рамках этого патча меняется **только визуальное представление главного экрана со снами**.

Не входит в scope:

* RAG
* vector DB
* retrieval
* embeddings
* backend processing pipeline
* механика анализа
* логика архетипов
* semantic search

Патч касается только:

* layout главного экрана
* визуальной структуры карточки сна
* поведения grid
* отображения title, date и gradient

---

## 2. Главный экран

### Было

Главный экран отображает сны как вертикальный список.

### Должно стать

Главный экран отображает сны как **grid из квадратных плиток**.

Визуальный ориентир:

* Instagram profile grid
* Pinterest-like gallery vibe, но более минималистично

---

## 3. Layout grid

Главный экран должен использовать:

* `GridView.builder`

Параметры:

* `crossAxisCount = 3`
* карточки квадратные
* одинаковые промежутки между карточками
* одинаковые внешние отступы по краям экрана

Рекомендуемые значения:

* spacing between tiles: `6–8 px`
* screen padding: `8–12 px`
* `childAspectRatio = 1`

---

## 4. Структура карточки сна

Каждая карточка сна должна содержать:

1. Фоновый градиент сна
2. Дату
3. Title сна

Карточка визуально воспринимается как цельная плитка.

---

## 5. Фон карточки

### 5.1 Если у сна есть анализ

Карточка должна быть полностью заполнена градиентом.

Градиент строится так:

* `color1 → color2 → background theme color`

Где:

* `color1` и `color2` — цвета, сохранённые у сна
* `background theme color` — фон текущей темы пользователя (light/dark)

Градиент должен заполнять **всю плитку**, а не только левую часть.

### 5.2 Если анализа нет

Если у сна ещё нет рассчитанного градиента:

* используется обычный фон текущей темы
* без дополнительной заливки

---

## 6. Дата

### Положение

Дата должна располагаться:

* **в верхней части карточки**
* с небольшим внутренним отступом

### Визуальный стиль

* шрифт маленький
* цвет контрастный к фону
* прозрачность выше обычного текста
* ориентир прозрачности: `60–70%`

Дата должна читаться, но не конкурировать с title.

### Формат

Использовать текущий формат проекта:

* дата + время

Пример:

* `20 Jan 2026, 08:14`
* либо тот формат, который уже используется в приложении

---

## 7. Title

### Положение

Title должен располагаться:

* **в нижней части карточки**
* поверх градиента

### Визуальный стиль

* контрастный цвет
* более крупный, чем дата
* жирнее даты
* максимум 2 строки
* overflow: ellipsis

### Ограничения

* отображать до 64 символов
* при переполнении обрезать многоточием

Title является главным визуальным элементом карточки.

---

## 8. Читаемость текста поверх градиента

Чтобы текст поверх градиента был читаемым, допускается:

* лёгкая тень текста
* лёгкий затемняющий overlay
* слабый градиент затемнения под текстом

Важно:

* не делать тяжёлую подложку
* не убивать лёгкий Instagram-style vibe
* текст должен быть читаем, но UI должен оставаться воздушным

---

## 9. Border radius

Карточки должны иметь мягкие скругления.

Рекомендуемый радиус:

* `12–16 px`

Карточки не должны быть остроугольными.

---

## 10. Поведение карточки

### Tap

Нажатие на карточку:

* открывает экран конкретного сна

### Long press

Долгое нажатие:

* открывает контекстное меню

Контекстное меню остаётся существующим, если оно уже реализовано.

В нём могут быть:

* редактировать
* изменить дату
* удалить

Данный документ не описывает изменение логики меню, только то, что плитки должны поддерживать long press.

---

## 11. Производительность

Так как у пользователя может быть много снов, grid должен быть реализован через ленивую подгрузку:

* `GridView.builder`

Недопустимо:

* рендерить все карточки сразу через статический список

---

## 12. Адаптация под тему

### Светлая тема

* background theme color = светлый фон приложения

### Тёмная тема

* background theme color = тёмный фон приложения

Градиент сна всегда должен корректно переходить в фон текущей темы пользователя.

То есть третий цвет градиента не жёстко белый, а:

* белый/светлый в light theme
* тёмный в dark theme

---

## 13. Компонентная структура

Рекомендуется выделить отдельный компонент карточки, например:

* `DreamTile`

Этот компонент должен принимать:

* `title`
* `date`
* `gradient colors`
* `theme background color`
* callbacks для tap и long press

Главный экран должен только строить grid и передавать данные в `DreamTile`.

---

## 14. Что не менять

В рамках этого патча не менять:

* backend API
* модель анализа
* механику генерации title
* механику вычисления gradient
* поиск
* profile
* RAG
* векторную БД
* chunk/retrieval logic

Если какие-то данные уже приходят с backend, они просто по-новому отображаются в grid UI.

---

## 15. Acceptance Criteria

Редизайн считается завершённым, если:

1. Главный экран показывает сны не списком, а сеткой 3xN.
2. Каждая карточка квадратная.
3. Градиент занимает всю карточку.
4. Дата отображается сверху карточки, маленькая и полупрозрачная.
5. Title отображается внизу карточки поверх градиента.
6. Карточка имеет мягкий border-radius.
7. Карточки открываются по tap.
8. Контекстное меню вызывается по long press.
9. UI корректно выглядит и в светлой, и в тёмной теме.
10. Grid работает через lazy rendering.

---

## 16. Итоговая визуальная модель

Карточка должна восприниматься как мини-постер сна.

Принцип:

* весь фон = настроение сна
* дата = лёгкая вспомогательная метка сверху
* title = главный смысловой якорь снизу

Общее ощущение интерфейса:

* Instagram grid
* clean
* dreamy
* minimal
* gallery of dreams
