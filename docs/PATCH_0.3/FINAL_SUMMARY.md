# JungAI PATCH 0.3 — Итоговая сводка реализации

Дата: 2026-03-13  
Статус: consolidated release notes  
Область: `client` + `backend` + `llm_service`

Этот документ объединяет изменения из:
- `AUTO_ANALYSIS.md`
- `IG_GRID.md`
- `ONBOARDING.md`
- `SPEECH_TO_TEXT.md`
- `RAG.md`
- `MAP.md`

## 1) AUTO_ANALYSIS — автоматический анализ после сохранения

Реализовано:
- Анализ запускается автоматически после `POST /api/v1/dreams`.
- На backend добавлен маппинг статусов анализа в ответы сна:
  - `saved`
  - `analyzing`
  - `analyzed`
  - `analysis_failed`
- На worker-уровне включен retry для временных LLM-ошибок (Celery autoretry/backoff).
- На клиенте добавлены визуальные состояния:
  - spinner в плитке при `analyzing`
  - fail-indicator при `analysis_failed`
- На экране сна доступен ручной fallback `Повторить анализ` для fail-state.

Технический результат:
- Базовый сценарий анализа полностью автоматический.
- Пользователь не обязан вручную запускать анализ нового сна.

## 2) IG_GRID — плиточный экран снов

Реализовано:
- Главный экран переведен на `GridView.builder`.
- Сетка 3 колонки, квадратные карточки.
- Карточка сна (`DreamCard`) показывает:
  - дату сверху
  - заголовок снизу
  - фон с градиентом (если анализ готов) или theme fallback.
- Поддержаны состояния `analyzing`/`analysis_failed` прямо в карточке.
- Сохранены действия:
  - tap -> открыть сон
  - long press -> контекстное меню.

Финальные UI-решения в реализации:
- Title ограничен до 32 символов в карточке.
- Title отображается до 3 строк, меньший размер, без жирного.
- В карточке даты отображается только дата (`dd.mm.yyyy`).

## 3) ONBOARDING — многошаговый модальный онбординг

Реализовано:
- Онбординг открывается модально поверх основного UI.
- Фон блюрится (modal overlay + `BackdropFilter`).
- Условие показа:
  - `onboarding_completed == false` или пустой `about_me`.
- После завершения:
  - `onboarding_completed = true`
  - `about_me` сохраняется.
- Реализованы 5 шагов.
- Шаг 1: пол + возраст
  - пол иконками (female/male/unknown)
  - возраст через dropdown.
- Остальные шаги — текстовые поля по ТЗ.

Финальные product-решения:
- Кнопка `Назад` убрана.
- Формирование `about_me` переведено в пользовательский формат:
  - без технических префиксов `Gender/Age/...`
  - значения на русском
  - разделитель `;`.

## 4) SPEECH_TO_TEXT — локальная запись + распознавание

Реализовано:
- На клиенте:
  - локальная запись аудио (record package),
  - индикация уровня записи,
  - отправка файла на backend,
  - вставка распознанного текста в поле ввода сна.
- На backend:
  - endpoint `POST /api/v1/audio/transcriptions`,
  - проксирование в CometAPI `/v1/audio/transcriptions`,
  - разбор transient/permanent ошибок.
- Модель транскрипции: `whisper-1` через CometAPI.

Итог:
- MVP работает как: локальная запись -> backend -> STT -> текст в форме.

## 5) RAG — переход к многослойной памяти

Реализовано:
- Chunk-based retrieval слой:
  - разбиение сна на chunks,
  - embeddings для chunks,
  - хранение и переиспользование.
- Structured analysis расширен:
  - `analysis_text` (Markdown),
  - `title`,
  - `gradient`,
  - `archetypes_delta`,
  - `symbol_entities`.
- Добавлен persistence для visual-слоя:
  - таблица `dream_symbol_entities`,
  - модель `DreamSymbolEntity`,
  - сохранение в pipeline `rebuild_dream_memory`.
- Raw symbols сохранены как отдельный внутренний слой (`dream_symbols`).
- В анализе и retrieval сохранена роль архетипов.

Итог:
- Retrieval foundation остался chunk-based.
- Визуальный semantic слой поднят до symbol entities.

## 6) MAP — карта символических сущностей

Реализовано:
- API карты:
  - `GET /api/v1/map/{user_id}`
  - `GET /api/v1/map/{user_id}/symbol/{symbol_id}`
  - `WS /api/v1/map/{user_id}/stream`
- Карта строится по `symbol_entities`, а не по raw chunk-фразам.
- В ответ map node включены:
  - `symbol_name`, `display_label`,
  - координаты `x/y/z`,
  - кластеры/цвет/веса,
  - `related_archetypes`.
- Детальная карточка symbol node:
  - связанные сны/фрагменты,
  - связанные архетипы,
  - related symbols.
- Archetype filters строятся как union всех `related_archetypes`, а не по одному доминирующему кластеру.
- Добавлена кнопка `Обновить` на карте:
  - пока идет пересчет, показывается предыдущая версия + индикатор обновления.
- Cache hotfix:
  - версия ключа карты поднята до `dream-map:v3`.

Финальное решение по UX карты (переопределено):
- Циклический wrap-around отключен.
- Карта ограничена одной областью (bounded pan, без toroidal прокрутки).

## 7) Качество symbol labels

Дополнительно ужесточено:
- фильтрация мусорных one-word символов,
- normalization на уровне LLM и backend,
- запрет для служебных/разговорных токенов,
- ориентация на человекочитаемые labels (2-3 слова).

Цель:
- убрать в карте и в detail значения типа `того`, `где`, `чуть`, `возможно`.

## 8) Что переопределено относительно исходных ТЗ

Зафиксированные product-изменения:
- ONBOARDING: без кнопки `Назад`.
- MAP: без циклической прокрутки.
- Symbol labels: фактический стандарт 2-3 слова (строже ранних версий ТЗ).

## 9) Миграция и эксплуатация

Для корректного применения PATCH 0.3:
1. Перезапустить `backend`, `celery_worker`, `llm_service`.
2. Убедиться, что авто-схема создала `dream_symbol_entities`.
3. Для старых снов выполнить повторный анализ (чтобы заполнить `symbol_entities` новым форматом).
4. При необходимости обновить карту кнопкой `Обновить`.

## 10) Итог PATCH 0.3

PATCH 0.3 перевел продукт в режим:
- автоматического анализа без ручного старта,
- более зрелого UI (grid + onboarding + голосовой ввод),
- многослойной памяти RAG,
- symbol-entity dream map с архетипическим фильтром и ручным контролем обновления.
